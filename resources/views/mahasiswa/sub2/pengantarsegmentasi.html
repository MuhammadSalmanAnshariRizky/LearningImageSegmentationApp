2. Pengantar Segmentasi Citra

Tujuan Pembelajaran
  • Menguraikan konsep segmentasi citra dengan menjelaskan pembagian citra menjadi beberapa wilayah beserta aturan-aturannya secara tepat.
  • Menguraikan metode segmentasi citra dengan menjelaskan perbedaan edge-based dan region-based secara sederhana.

2.1 Segmentasi Citra

Segmentasi citra merepresentasikan seluruh wilayah spasial yang ditempati oleh suatu citra. Ruang spasial (spatial domain) adalah ruang dua dimensi yang terdiri dari seluruh koordinat posisi piksel yang membentuk suatu citra. Dalam pengolahan citra, ruang spasial merepresentasikan lokasi geometris piksel Lihat gambar untuk mengetahui apa itu wilayah spasial.

Gambar 2.1 Visualisasi ruang spasial

Segmentasi citra adalah proses membagi seluruh area gambar (R) menjadi beberapa bagian kecil (R₁, R₂, ..., Rₙ), di mana setiap bagian mewakili suatu wilayah (n) tertentu dalam gambar. sehingga:

Aturan Pertama

\[ \bigcup_{i=1}^{n} R_i = R \]

Misalkan n = 5, maka dapat dituliskan secara eksplisit sebagai berikut :

\[ R_1 \cup R_2 \cup R_3 \cup R_4 \cup R_5 = R \]

Gambar 2.2 Visualisasi aturan pertama dari segmentasi

Jika semua subwilayah digabungkan kembali, maka hasilnya adalah seluruh citra. Tidak ada satu piksel pun yang tertinggal di luar pembagian tersebut.

Aturan Kedua

Rᵢ adalah himpunan yang terhubung (connected set), untuk i = 1,2,...,n.

Artinya Adalah, Setiap wilayah hasil segmentasi harus berupa satu bagian yang menyatu, tidak terpisah-pisah. Dengan kata lain:

\[ \text{Semua piksel dalam setiap } R_i \text{ harus saling terhubung.} \] \[ \text{Artinya, } R_1, R_2, R_3, R_4, \text{ dan } R_5 \text{ masing-masing harus membentuk satu wilayah yang terhubung.} \]

Gambar 2.3 Visualisasi aturan kedua dari segmentasi

Tidak boleh ada bagian kecil yang terpisah jauh tetapi masih dianggap satu wilayah (region). Lihatlah gambar 2.4 dibawah ini

Gambar 2.4 Visualisasi contoh wilayah yang terpisah jauh

Dalam citra digital, keterhubungan biasanya didefinisikan sebagai :

  1. 4 - connected
    Dua piksel dikatakan 4 - connected jika terhubung melalui piksel atas, bawah, kiri, dan kanan.

  2. 8 - connected
    Dua piksel dikatakan 8 - connected jika terhubung melalui piksel atas, bawah, kiri, kanan dan 4 arah diagonal.

Gambar 2.5 Konsep ketetanggan (keterhubungan)

Aturan Ketiga

\[ R_i \cap R_j = \emptyset \quad \text{untuk semua } i \neq j \]

Artinya, tidak boleh ada piksel yang menjadi anggota dua wilayah (region) sekaligus. Dengan kata lain:

  • R₁ dan R₂ tidak boleh saling tumpang tindih. Contoh dibawah ini adalah wilayah yang saling tumpang tindih atau beririsan

    Gambar 2.6 Visualisasi aturan ketiga dari segmentasi

  • Jika satu piksel sudah masuk R₁, dia tidak boleh masuk R₂. Lihatlah pada gambar 2.3

Aturan Keempat

\[ Q(R_i) = TRUE \quad \text{untuk } i = 1,2,\ldots,n \]

Artinya adalah Setiap wilayah (region) Ri harus memenuhi suatu kriteria atau syarat tertentu.

Keterangan :

  • Q = predikat logika (aturan/kriteria)
  • TRUE = syaratnya terpenuhi

Q adalah aturan yang kita tentukan sendiri. Contoh paling umum dalam segmentasi ialah sebagai berikut :

  • Berdasarkan intensitas
  • Berdasarkan standar deviasi
  • Berdasarkan warna

Contoh :

\[ Q(R_i) = TRUE \text{ jika semua piksel dalam } R_i \text{ memiliki nilai intensitas yang sama} \] \[ Q(R_1) = TRUE \text{ jika } f(x,y) = 1 \text{ untuk semua } (x,y) \in R_1 \] \[ Q(R_2) = TRUE \text{ jika } f(x,y) = 2 \text{ untuk semua } (x,y) \in R_2 \] \[ Q(R_3) = TRUE \text{ jika } f(x,y) = 3 \text{ untuk semua } (x,y) \in R_3 \] \[ Q(R_4) = TRUE \text{ jika } f(x,y) = 4 \text{ untuk semua } (x,y) \in R_4 \] \[ Q(R_5) = TRUE \text{ jika } f(x,y) = 5 \text{ untuk semua } (x,y) \in R_5 \]

Hasil Analisis Tiap Region

Nilai dalam R₁:

{1, 1, 1, 1, 1}

Karena semua sama:

\(Q(R_1) = TRUE\)

Nilai dalam R₂:

{2, 2, 2, 2}

Semua sama:

\(Q(R_2) = TRUE\)

Nilai dalam R₃:

{3, 6, 3, 6, 3, 7}

Karena tidak semua sama:

\(Q(R_3) = FALSE\)

Nilai dalam R₄:

{4, 4, 4, 4, 4, 4}

\(Q(R_4) = TRUE\)

Nilai dalam R₅:

{5, 5, 5, 5}

\(Q(R_5) = TRUE\)

Karena:

\(Q(R_3) = FALSE\)

Maka syarat:

\(Q(R_i) = TRUE \ \mathrm{untuk\ semua\ } i\)

Artinya segmentasi belum valid jika menggunakan kriteria homogenitas intensitas. Karena pada R3 tidak memenuhi predikat homogenitas yang sudah ditentukan
Ingat bahwa
Setiap wilayah (region) harus memiliki karakteristik yang seragam. Jika satu wilayah (region) memiliki piksel yang sangat berbeda dari yang lain, maka wilayah (region) tersebut tidak memenuhi kriteria segmentasi.

Aturan Kelima

\[ Q(R_i \cup R_j) = FALSE \quad \text{untuk setiap pasangan wilayah bertetangga } R_i \text{ dan } R_j \]

Artinya, jika dua wilayah (region) yang bertetangga harus berbeda karakteristiknya. Jika digabung dan masih seragam, berarti sebenarnya mereka satu wilayah (region) yang sama.

Contohnya :

Misalnya ada dua region bertetangga:

\[ Q(R_1) = TRUE \text{ jika } f(x,y) = 1 \text{ untuk semua } (x,y) \in R_1 \] \[ Q(R_2) = TRUE \text{ jika } f(x,y) = 2 \text{ untuk semua } (x,y) \in R_2 \]

Region 1

\(R_1 = \{1,1,1,1\}\)

Region 2

\(R_2 = \{2,2,2,2\}\)

Karena masing-masing homogen:

\(Q(R_1) = TRUE\)
\(Q(R_2) = TRUE\)

Karena ada dua nilai berbeda, maka:

\(R_1 \cup R_2 = \{1,1,1,1,2,2,2,2\}\)

\(Q(R_1 \cup R_2) = FALSE\)
