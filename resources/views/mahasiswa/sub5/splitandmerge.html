5.3 Region Splitting and Merging

Kombinasi antara region splitting dan region merging dapat menghasilkan metode yang memiliki keunggulan dari keduanya. Pendekatan split-and-merge bekerja menggunakan representasi citra berbentuk piramida (pyramid), di mana region berbentuk persegi dan berada pada tingkat resolusi tertentu.

Gambar 5.3 Visualisasi proses dari split and merge

Berdasarkan ilustrasi gambar tersebut, jika suatu region pada tingkat piramida tidak memenuhi kriteria homogenitas (kecuali pada tingkat paling bawah), maka region tersebut akan dipecah menjadi empat subregion. Subregion ini merupakan representasi dengan resolusi yang lebih tinggi pada tingkat di bawahnya.

Sebaliknya, jika terdapat empat region pada suatu tingkat yang memiliki nilai homogenitas yang hampir sama, maka region-region tersebut dapat digabung menjadi satu region pada tingkat yang lebih tinggi.

Proses segmentasi ini dapat dipahami sebagai pembentukan struktur pohon yang disebut segmentation quadtree, di mana setiap daun (leaf node) merepresentasikan satu region yang homogen. Proses splitting dan merging berkaitan dengan pembentukan atau penghapusan bagian dari pohon tersebut. Jumlah node daun pada pohon menunjukkan jumlah region hasil segmentasi akhir. Berikut adalah ilustrasi dari segmentation quadtree:

Gambar 5.4 Contoh ilustrasi region splitting dan struktur data quadtree

Metode split-and-merge biasanya menggunakan struktur data seperti region adjacency graph untuk menyimpan informasi keterhubungan antar region. Penggunaan struktur pohon segmentasi ini memudahkan implementasi dan meningkatkan efisiensi komputasi, meskipun region yang dihasilkan tidak selalu saling terhubung secara langsung.

Salah satu kelemahan metode ini adalah asumsi bahwa region berbentuk persegi. Oleh karena itu, sering ditambahkan proses tambahan untuk memungkinkan penggabungan region yang tidak berada pada cabang yang sama dalam pohon segmentasi.

Region awal dapat ditentukan secara bebas atau berdasarkan informasi awal (prior knowledge). Karena metode ini menggabungkan proses splitting dan merging, maka segmentasi awal tidak harus langsung memenuhi kondisi:

\[ Q(R_i) = \text{TRUE} \quad \text{atau} \quad Q(R_i \cup R_j) = \text{FALSE} \]

Kriteria homogenitas tetap menjadi faktor utama dalam metode ini. Untuk citra sederhana, kriteria ini dapat didasarkan pada sifat lokal citra. Namun, untuk citra yang lebih kompleks, bahkan kriteria yang lebih lengkap sekalipun belum tentu menghasilkan segmentasi yang optimal.

Algoritma Split and Merge

Berikut adalah langkah-langkah detail dari penerapan algoritma split and merge:

Langkah 1
Inisialisasi Awal:
Tentukan segmentasi awal citra menjadi beberapa region, kriteria homogenitas \( Q(R_i) \), serta struktur data pyramid.

Langkah 2
Pengecekan Kondisi Suatu Region (Splitting & Merging):
Jika terdapat suatu region \( R \) pada struktur pyramid yang tidak homogen atau bernilai \[ Q(R) = \text{FALSE} \] maka region tersebut wajib dipecah menjadi empat subregion terpisah.

Sebaliknya, jika terdapat empat region dengan parent yang sama dan memenuhi kriteria gabungan:

\[ Q(R_1 \cup R_2 \cup R_3 \cup R_4) = \text{TRUE} \]
maka keempat region tersebut digabung menjadi satu region homogen. Jika tidak ada lagi region yang dapat di-split atau di-merge, lanjut ke langkah 3.

Langkah 3
Pengecekan Dua Region yang Bertetangga (Adjacent Merging):
Jika terdapat dua region bertetangga \( R_i \) and \( R_j \) (meskipun berada pada level pyramid yang berbeda atau tidak memiliki parent yang sama) dan terbukti memenuhi kondisi:

\[ Q(R_i \cup R_j) = \text{TRUE} \]
maka kedua region bertetangga tersebut digabung menjadi satu region utuh.

Langkah 4
Penggabungan Tambahan (Eliminasi Region Kecil):
Gabungkan region berukuran kecil dengan region tetangga yang paling mirip jika diperlukan untuk menghilangkan pecahan region yang terlalu kecil (spurious regions).

def is_homogeneous(region):
    sigma = np.std(region)
    mean = np.mean(region)
    return (
        sigma < STD_THRESHOLD
    )

Fungsi is_homogeneous() digunakan untuk menentukan apakah suatu region bersifat homogen. np.std(region) digunakan untuk menghitung standar deviasi intensitas piksel pada region. np.mean(region) digunakan untuk menghitung rata-rata intensitas piksel pada region. label_counter = 1
def split_region(img, labels, x, y, w, h):
    global label_counter
    region = img[y:y+h, x:x+w]
    if (
        w <= MIN_SIZE
        or h <= MIN_SIZE
        or is_homogeneous(region)
    ):
        labels[y:y+h, x:x+w] = label_counter
        label_counter += 1
        return
    hw = w // 2
    hh = h // 2
    split_region(img, labels, x, y, hw, hh)
    split_region(img, labels, x + hw, y, w - hw, hh)
    split_region(img, labels, x, y + hh, hw, h - hh)
    split_region(img, labels, x + hw, y + hh, w - hw, h - hh)

Tahap ini berfungsi membagi region yang tidak homogen menjadi empat subregion. Program terlebih dahulu memeriksa apakah ukuran region sudah terlalu kecil atau region sudah homogen. Jika salah satu kondisi tersebut terpenuhi, proses splitting dihentikan dan region diberi label. Jika region belum homogen, maka region dibagi menjadi empat bagian menggunakan pendekatan quadtree. Proses splitting dilakukan secara rekursif hingga seluruh region memenuhi kriteria homogenitas atau mencapai ukuran minimum.

def merge_regions(img, labels):
    changed = True
    while changed:
        changed = False
        unique_labels = np.unique(labels)
        for label1 in unique_labels:
            mask1 = labels == label1
            if np.sum(mask1) == 0:
                continue
            mean1 = np.mean(img[mask1])
            dilated = cv2.dilate(
                mask1.astype(np.uint8),
                np.ones((3,3), np.uint8)
            )
            neighbors = np.unique(labels[dilated > 0])
            for label2 in neighbors:
                if label1 == label2:
                    continue
                mask2 = labels == label2
                if np.sum(mask2) == 0:
                    continue
                mean2 = np.mean(img[mask2])
                if abs(mean1 - mean2) < MERGE_THRESHOLD:
                    labels[mask2] = label1
                    changed = True
    return labels

Tahap ini digunakan untuk menggabungkan region-region bertetangga yang memiliki karakteristik serupa. Program mencari seluruh label region yang telah terbentuk dari proses splitting. Rata-rata intensitas masing-masing region dihitung sebagai dasar penggabungan. Fungsi cv2.dilate() digunakan untuk menemukan region tetangga. Dua region akan digabung apabila selisih rata-rata intensitasnya lebih kecil dari nilai MERGE_THRESHOLD. Proses penggabungan dilakukan berulang hingga tidak ada lagi region yang memenuhi syarat untuk di-merge.

def create_binary_segmentation(img, labels):
    output = np.zeros_like(img)
    unique_labels = np.unique(labels)
    for label in unique_labels:
        mask = labels == label
        mean_intensity = np.mean(img[mask])
        if mean_intensity < 120:
            output[mask] = 255
    return output

Tahap ini digunakan untuk membentuk citra hasil segmentasi. Program membuat citra kosong yang akan digunakan untuk menyimpan hasil akhir. Setiap region diperiksa berdasarkan rata-rata intensitasnya. Region yang memenuhi kriteria objek akan diberi nilai 255 (putih). Region lainnya akan tetap bernilai 0 (hitam). Hasil akhirnya berupa citra biner yang menampilkan objek dan background.
